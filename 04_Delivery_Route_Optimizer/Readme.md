# 🚚 배송 경로 최적화 시뮬레이터 (Python-Route-Optimizer)

**[핵심 요약]**  
경험과 감에 의존하던 배차 및 라우팅 업무를 데이터 기반으로 시뮬레이션하기 위해 개발한 **Python 기반 배송 경로 최적화 프로젝트**입니다.  
주소 데이터만 입력하면 카카오 API와 TSP(최근접 이웃 알고리즘)를 활용해 최적 경로, 예상 도착시간, 유류비까지 자동으로 산출합니다.  
**AI를 적극 활용하여 아이디어를 실제 동작하는 코드로 구현하는 개발 역량을 기르기 위해 진행한 프로젝트입니다.**

---

## 🔍 배경 및 문제 정의

매일 반복되는 배송 및 배차 업무는 담당자의 경험에 전적으로 의존하는 경향이 컸습니다.  
이러한 직관 기반의 업무 방식을 데이터 기반으로 시뮬레이션하기 위해 다음과 같은 개선점들을 정의했습니다.

| 업무 | 기존 방식 (Pain Point) | 시뮬레이터 적용 후 (Solution) |
| :--- | :--- | :--- |
| **배송 경로 결정** | 담당자 경험 기반 수동 배정 | 최근접 이웃 알고리즘으로 자동 최적화 |
| **차량별 업무 불균형** | 특정 차량에 배송 과부하 발생 | 자동 밸런싱 로직으로 균등 배분 |
| **예상 도착시간** | 감으로 어림잡아 전달 | 평균 속도 및 서비스 시간 기반 자동 계산 |
| **유류비 산출** | 별도 수작업 계산 | 차량별 연비 적용 자동 계산 |
| **거점 이전 시뮬레이션** | 물리적 제약으로 시도조차 어려움 | 출발지 변수 변경만으로 즉시 재시뮬레이션 |

---

## 🔄 해결 방법 (Workflow)

> **[데이터 입력]** 주소 데이터 입력 (`배차데이터_양식.xlsx`)  
> **↓**  
> **[좌표 변환]** `geocoding.py` 실행 → 카카오 API로 주소를 위경도 좌표로 자동 변환  
> **↓**  
> **[1차 배정]** `fixed_route_simulation.py` 실행 → 차량별 담당 구역 키워드로 1차 배정  
> **↓**  
> **[경로 최적화]** 최근접 이웃(Nearest Neighbor) 알고리즘으로 이동 동선 최적화  
> **↓**  
> **[밸런싱]** 강제 밸런싱 및 미세 조정으로 차량별 업무량 균등화  
> **↓**  
> **[결과 출력]** Folium 지도 시각화 HTML 및 상세 배차표/유류비 통계 엑셀 생성

## 🛠 주요 기능

### 1. 📍 주소 → 좌표 자동 변환 (`geocoding.py`)
카카오 로컬 API를 활용하여 엑셀의 주소 데이터를 위경도 좌표로 일괄 변환합니다.
- SAP 헤더명을 프로그램 변수로 자동 매핑
- 변환 진행 상황 실시간 터미널 출력
- 변환 실패 건 자동 표시 (검색 실패 / API 에러 구분)

### 2. 🛣️ 최적 경로 자동 배정 (`fixed_route_simulation.py`)
- 차량별 담당 구역 키워드 기반 1차 자동 배정
- 최근접 이웃 알고리즘(TSP)을 통한 경로 최적화 연산
- 차량 간 업무 불균형 자동 감지 및 재배분
- 전체 미세 조정으로 최종 균등화

### 3. ⏱️ 예상 도착시간 및 ⛽ 유류비 자동 계산
- **도착시간:** 평균 속도(35km/h)와 거래처당 서비스 대기 시간(5분)을 기반으로 각 배송지의 예상 도착시간(ETA)을 자동 산출합니다.
- **유류비:** 차량별 연비와 총 주행거리를 바탕으로 예상 유류비를 계산하며, 유류 단가 및 연비 설정은 환경에 맞게 변경 가능합니다.

### 4. 🗺️ Folium 지도 시각화
차량별 배송 경로를 색상으로 직관적으로 구분하여, 브라우저에서 확대/축소가 가능한 인터랙티브 HTML 지도로 렌더링합니다.

## 📁 파일 구조 및 실행 환경

```text
Python-Route-Optimizer/
├── geocoding.py                  # 주소 → 좌표 변환 (카카오 API)

```

최종_배차결과_통계포함.xlsx          # 상세 배차표 + 배차요약(유류비 포함)

### ✨ 장점
출발지 변수나 권역 데이터만 수정하면 무한한 'What-If' 시뮬레이션이 가능합니다.

### 🚀 기대 효과

### ⚠️ 주의사항 (PoC 모델)
본 프로젝트는 실제 현장 운영에 바로 도입하기 위한 것이 아닌, 알고리즘을 통한 배송 코스의 효율성을 확인하고 시뮬레이션하기 위한 **개념 증명(PoC) 모델**입니다.
