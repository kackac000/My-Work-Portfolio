# 📷 SAP 연동 지능형 바코드 스캐너 (CSharp-SAP-Scanner)

현장에서 직접 겪은 병목 현상을 소프트웨어로 해결한 C# 기반 RPA(Robotic Process Automation) 프로젝트입니다.
하드웨어 교체나 시스템 권한 우회 없이, **순수 소프트웨어만으로 바코드 스캔 오류와 수기 입력 문제를 완벽하게 제거**했습니다.

---

## 🔍 배경 및 문제 정의
의약품 물류 현장에서 SAP EWM 시스템으로 입출고 검수를 진행할 때, 다음과 같은 고질적인 문제들이 반복적으로 발생했습니다.

| 문제 | 원인 | 기존 해결 방법 | 개선 효과 |
| :--- | :--- | :--- | :--- |
| **묶음 바코드 미인식** | SAP가 GS1 복합 바코드 포맷을 기본 지원하지 않음 | 낱개 바코드를 하나씩 수동으로 스캔 | **80% 시간 단축** (5초 → 1초) |
| **제약사별 바코드 오생성** | 일부 제조사의 GS1 표준 미준수 및 독자 규격 사용 | 담당자가 직접 판단하여 포맷을 변환 후 수동 처리 | 로직에 의한 자동 보정 |

---

## ✅ 해결 방법 (Workflow)
SAP GUI 자동화, 패턴 기반 바코드 파싱 엔진, 그리고 컴퓨터 비전(OpenCV) 기술을 결합하여 비침투적(Non-invasive) 방식의 데스크톱 프로그램을 개발했습니다.

> **[바코드 스캔]** 스캐너를 통해 텍스트박스로 raw 데이터 입력 (Enter 트리거)
> **↓**
> **[파싱 엔진]** `BarcodeParser`가 바코드 종류를 자동 식별하고 전송 규격에 맞게 포맷 변환
> **↓**
> **[UI 추적]** `OpenCvSharp`의 템플릿 매칭(`CCoeffNormed`)으로 SAP 화면 내 정확한 입력 필드 좌표 탐색
> **↓**
> **[자동 입력]** `WinAPI`를 호출하여 백그라운드 SAP 창으로 포커스 이동 후, 클립보드 데이터 붙여넣기 및 Enter 전송
> **↓**
> **[자동 저장]** 검수 완료 화면(`complete.png`) 감지 시 자동으로 F11(저장) 단축키 실행

## 🛠 주요 기능

* **1. 바코드 자동 분류 및 변환**
스캔된 바코드를 실시간으로 분석하여 5가지 타입으로 자동 분류합니다.
| 타입 | 설명 |
| :--- | :--- |
| **OBD** | 출고 전표 바코드 (8~10자리 특정 시작 번호) |
| **GENERAL** | 개별 바코드 (단순 및 제조/유효/시리얼이 포함된 복잡 포맷 지원) |
| **GENERAL_SPECIAL** | SAP 팝업 창 상태 유지가 필요한 특수 길이 바코드 |
| **BUNDLE** | 묶음 포장 바코드 (단순 및 복잡 포맷 지원) |
| **UNKNOWN** | 미분류 예외 처리 (원문 그대로 전송하여 중단 방지) |

* **2. 제약사 오생성 바코드 자동 보정**
일부 제약사가 GS1 표준을 잘못 적용하여 생성한 바코드를 소프트웨어 레벨에서 패턴을 인식해 자동으로 보정합니다. (하드웨어나 제약사 시스템 변경 불필요)

* **3. OpenCV 이미지 인식으로 SAP 필드 자동 탐색**
SAP GUI는 외부 자동화 API 연결이 불가한 환경이므로, 화면 캡처와 이미지 템플릿 매칭을 결합하여 동적으로 입력 필드의 X, Y 좌표를 찾아냅니다. 해상도 변화나 창 위치 이동에도 유연하게 대응합니다.

* **4. 검수 완료 자동 저장**
검수 완료 팝업이나 화면 상태를 이미지로 감지하면 작업자의 개입 없이 자동으로 `F11`(저장) 이벤트를 발생시켜 작업 사이클을 단축합니다.

* **5. 중복 실행 방지 및 화면 자동 배치**
프로그램 시작 시 프로세스 중복 실행을 차단하고, SAP 주 작업 창 옆으로 스캐너 UI를 자동 배치(`SetWindowPos`)하여 작업 흐름을 방해하지 않습니다.

## ⚙️ 실행 환경 및 필수 설정

* **OS:** Windows 10 / 11
* **Runtime:** .NET 8 Desktop Runtime
* **Environment:** SAP GUI가 실행 중인 데스크톱 환경

**💡 필수 템플릿 이미지 (Calibration)**
실행 파일과 동일한 경로에 화면 인식을 위한 아래의 템플릿 이미지가 **반드시 필요**합니다. 
*(보안 및 저작권 문제로 본 저장소에는 샘플 이미지가 포함되어 있지 않습니다. 현장 모니터 해상도에 맞춰 직접 캡처하여 사용해야 합니다.)*

* `Item.png` : 입고 검수 화면의 텍스트 입력 필드
* `ERPdoc.png` : 출고 검수 화면의 텍스트 입력 필드
* `complete.png` : 검수 완료 상태 감지용 팝업 또는 UI 영역

## 📝 개발 배경 노트

이 프로그램은 **15년간 의약품 물류 현장에서 직접 겪은 문제를 해결하기 위해** 만들었습니다.

SAP 시스템 운영 책임자로서 거대한 ERP 시스템이 가진 구조적 한계와 현장의 괴리를 누구보다 정확히 파악하고 있었기에, 무거운 외부 시스템 연동이나 거창한 인프라 도입 없이 **"현장에서 즉시, 확실하게 작동하는 방식"**으로 설계했습니다.

단순히 코드를 짜는 것을 넘어, **"무엇을 어떻게 만들어야 실제 작업자의 불편을 해소할 수 있는가"**에 대한 치열한 고민의 결과물입니다.
