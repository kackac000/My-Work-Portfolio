## 1. SAP 연동 바코드 스캐너 (C#)

기존 SAP 시스템의 바코드 인식 한계를 하드웨어가 아닌 **소프트웨어 기술로 극복**하여, 현장 작업 속도를 획기적으로 개선한 프로젝트입니다.

- **개발 동기:** SAP 시스템에서 묶음 바코드나 오염된 바코드가 인식되지 않아 발생하는 수기 입력의 비효율과 휴먼 에러를 해결하고자 개발했습니다.
- **주요 기술:** C# WinForms, OpenCVSharp (이미지 템플릿 매칭), WinAPI (외부 프로그램 제어)
- **핵심 기능:**
    - **지능형 파싱:** 스캔된 바코드의 패턴을 분석하여 종류(개별, 묶음, OBD 등)를 자동으로 식별.
    - **자동 입력 자동화:** 특정 바코드를 인식 가능한 형태로 변환하고, 이미지 인식을 통해 SAP 화면의 정확한 필드에 값을 입력 및 엔터 처리까지 자동 수행.

## 2. 엑셀 업무 자동화 스크립트 (VBA)

매일 반복되는 데이터 가공 업무를 원클릭으로 처리하여, 관리자가 **데이터 분석과 전략 수립 등 핵심 업무에 집중**할 수 있도록 만든 툴입니다.

- **개발 동기:** 단순 반복적인 엑셀 작업에 소요되는 시간을 줄이고, 데이터 정합성을 확보하기 위해 개발했습니다.
- **주요 기능:**
    - **파일 자동 분할:** 마스터 파일을 거래처, 권역 등 특정 기준에 맞춰 수십 개의 개별 파일로 자동 분할 및 저장.
    - **문서 자동 생성:** 원본 데이터를 기반으로 정해진 양식의 재고 조사 리스트 및 PO(발주서)를 즉시 생성하여 문서 작업 시간 90% 단축.

## 3. OSRM 기반 배송 경로 최적화 및 시뮬레이터 (Python & AI)

경험과 감에 의존하던 배차 업무를 **데이터 기반(Data-Driven)**으로 전환하고, 물류 거점 통폐합 등 변화하는 환경에 민첩하게 대응하기 위해 개발한 최신 프로젝트입니다.

- **개발 동기:** 물류 센터 통합이나 배송 권역 변경 시나리오 발생 시, 예상되는 이동 거리와 비용 효율을 사전에 검증하고 최적의 라우팅을 도출하기 위함.
- **주요 기술:** Python, OSRM (Open Source Routing Machine), Docker, Pandas, Folium
- **AI 협업 (AI-Assisted Development):**
    - **생성형 AI 적극 활용:** 최신 기술 스택(Docker, OSRM) 도입 과정에서 AI를 코딩 파트너로 활용하여 개발 시간을 단축하고, 복잡한 알고리즘 구현의 정확도를 높임.
    - **신기술 수용력 증명:** 익숙한 C#/VBA 환경에 안주하지 않고, AI를 도구 삼아 Python 기반의 데이터 분석 환경을 빠르게 구축함.
- **핵심 기능:**
    - **경로 최적화(VRP):** 수백 개의 배송지를 분석하여 최단 거리 및 최적 이동 경로 자동 산출.
    - **물류 시뮬레이션:** 출발지(센터) 변경에 따른 총 운행 거리 및 소요 시간 변화를 즉시 시뮬레이션하여 거점 전략 수립 근거 제공.
    - **데이터 시각화:** 텍스트 데이터로만 존재하던 배송 경로를 지도(Map) 위에 시각화하여 직관적인 현황 파악 지원.

## 4. 💊 의약품 유통 분석 및 AI 수요 예측 시스템

950만 건의 대규모 의약품 유통 데이터를 처리하고, 현업 비즈니스 로직(약가인하 분류 등)을 반영한 대시보드 프로젝트입니다.

## ✨ 핵심 기능
- **고속 데이터 처리:** DuckDB를 활용한 대용량 데이터(950만 건)의 초고속 분석 환경 구축
- **자동화된 ETL:** 엑셀 데이터를 DB로 자동 변환 및 약가인하(601+1006) 자동 분류 로직 구현
- **영업 기회 예측:** 하이브리드 알고리즘(최근 트렌드 + 계절성) 기반 추가 발주 가능량 산출
- **UI/UX:** Streamlit을 활용한 직관적인 지점별 KPI 시각화

## 🛠 Tech Stack
- **Language:** Python 3.13
- **Database:** DuckDB (OLAP 최적화)
- **Visualization:** Streamlit, Plotly
